# 项目文件说明（面向不熟悉深度学习代码的阅读顺序）

下面按 **「建议阅读顺序」** 排列，并说明每个文件**在实验里扮演的角色**。  
代码里已逐步补充 **中文注释**；若某段仍看不懂，可先对照本表抓住「输入 / 输出 / 和谁配合」。

---

## 一、建议阅读顺序（由浅入深）

1. **`文件说明.md`**（本文件）— 地图  
2. **`pipeline.py`** — 数据从原图怎么变成 1024 patch  
3. **`dataset_seg.py`** — 分割数据集里一张图怎么变成张量（给旧流程或增强用）  
4. **`dataset_seg_yolo.py`** — 语义 mask 怎么改成 YOLO-seg 要的「实例」格式  
5. **`dataset_det_from_masks.py`** — 语义 mask 怎么改成检测框  
6. **`models/teacher_vit.py`** — 教师 ViT 怎么抽多层特征（本地 HF 快照 + `transformers`）  
7. **`distill_modules.py`** — 通道对齐、教师相邻层融合、NWD、特征 L1 等**可复用积木**  
8. **`train.py`** — 分割训练主循环（YOLO-seg + 可选蒸馏）  
9. **`detect_nwd_distill_loss.py`** — 检测损失：BCE + NWD + DFL + 蒸馏  
10. **`train_detect_distill.py`** — 检测训练主循环  

工具脚本（与训练无关时可最后看）：**`scripts/hf_login.py`**、**`scripts/download_dinov3_teacher.py`**。

## 三、`solution/` 核心代码（按文件名）

### `pipeline.py` — 数据预处理与切片

- **输入**：原始 `images/`、`masks/`（整图或大场景）  
- **输出**：`output_dir/images/`、`masks/`、`meta/` 下的 **固定尺寸 patch**（默认 1024×1024）  
- **要点**：仅**滑窗**切片 + 原图缩放与越界 padding；**图与 mask 同步几何变换**（原图缩放时 mask 用最近邻）  
- **你不写训练时也要先跑它**，否则后面 `train*.py` 没有数据  

### `dataset_seg.py` — 语义分割 Dataset（像素级）

- **作用**：读成对 PNG，`__getitem__` 返回 `image`（CHW，ImageNet 归一化）、`mask`（长整型类别 id）  
- **谁用**：与 **轻量自定义分割网络** 搭配；当前主流程 **`train.py` 已改用 YOLO-seg**，但本文件仍提供 **`split_stems`**、**`default_train_transforms`**（随机翻转）给 `dataset_seg_yolo` 使用  

### `dataset_seg_yolo.py` — 语义 mask → YOLO 实例分割标签

- **作用**：把「每个像素一个类」的图，用 **连通域** 拆成多个实例，生成 `batch_idx / cls / bboxes / merged_instance_mask`  
- **谁用**：**`train.py`**（Ultralytics `v8SegmentationLoss`）  

### `dataset_det_from_masks.py` — 语义 mask → 检测框

- **作用**：对每个前景类做连通域，外接框 → YOLO 格式 **cx,cy,w,h 归一化**  
- **谁用**：**`train_detect_distill.py`**  

### `models/teacher_vit.py` — 冻结教师网络

- **`HFViTTeacher`**：用 **transformers** 对**本地目录**做 `from_pretrained`（目录须含 `config.json`），`output_hidden_states` 取第 3/4/7/8/11/12 层并对齐 DINOv3 的 patch 序列  
- **`build_teacher`**：默认从 **`solution/weights/dinov3-vitb16-pretrain-lvd1689m/`** 加载；可通过 **`--teacher-weights`** 指定其它本地快照目录  
- **`default_teacher_weights_dir`**：返回上述默认权重目录的 `Path`（供脚本或测试使用）  

### `models/__init__.py`

- 导出 `build_teacher`、`HFViTTeacher`、`default_teacher_weights_dir`  

### `distill_modules.py` — 蒸馏公共模块

- **`StudentChannelAlign`**：把学生 P3/P4/P5 的通道 **1×1 卷积** 投到与教师相同维度（默认 768）  
- **`AdaptiveTeacherFusion`**：教师相邻两层 **先对齐分辨率再学权重融合**  
- **`FeatureDistillLoss`**：多层 **SmoothL1** 特征蒸馏  
- **`NWDLoss` / `total_loss`**：给检测或自定义循环用的 **NWD 与加权和**（检测里主要用 `detect_nwd_distill_loss` 封装）  

### `train.py` — YOLO 分割训练 + 可选特征蒸馏

- **学生**：Ultralytics **SegmentationModel**（如 `yolo11m-seg.pt`）  
- **损失**：`v8SegmentationLoss`（含 box、seg、cls、dfl）  
- **蒸馏**：hook **Segment 头输入** 的三层特征 → `align` → 与教师融合特征算 L_feat  
- **命令行**：`--data-dir`、`--student-weights`、`--imgsz`、`--distill`、`--teacher-weights`（可选，默认同上本地目录）等  

### `detect_nwd_distill_loss.py` — 检测用自定义损失类

- **作用**：在 Ultralytics 检测头上，把 **CIoU 框损失换成 NWD**，保留 **DFL、分类 BCE**，并加 **特征蒸馏**  
- **类 `NWDDetectionDistillLoss`**：内部注册 hook 取 Detect 输入特征，与教师对齐  

### `train_detect_distill.py` — 检测训练主程序

- **数据**：`dataset_det_from_masks` + `collate_det`  
- **学生**：`YOLO(...).model`  
- **优化**：学生 + `StudentChannelAlign` + `AdaptiveTeacherFusion`；教师冻结

## 六、深度学习里几个「行话」对照（读代码时有用）

- **张量 Tensor**：可理解为多维数组；`[B,3,H,W]` 常表示一批图像  
- **Dataset / DataLoader**：Dataset 定义「第 i 个样本怎么读」，DataLoader 负责批大小、打乱、多进程  
- **forward / 前向**：数据从网络输入算到输出（和 **backward 反传** 相对）  
- **冻结**：`requires_grad=False`，不参与更新，本项目中 **教师默认冻结**  
- **蒸馏**：让学生模仿教师的中间层特征（或输出），额外加一项 **L_feat**
