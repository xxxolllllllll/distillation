- **2 异构蒸馏方法**

  - **2.1 异构网络构建与层级对应关系**

  - 为充分利用视觉大模型的表征能力，本文构建了由教师模型与学生模型组成的异构蒸馏框架。教师模型采用 facebook/dinov3-vitb16-pretrain-lvd1689m，学生模型采用 YOLOv11-m。训练与推理阶段输入图像统一为 1024×1024，并采用保持纵横比的缩放与 padding 预处理方式，以减少几何变形对缺陷形态的影响。在层级监督设计上，学生模型输出三层特征 P3/P4/P5，其空间分辨率分别为 128×128、64×64、32×32。教师模型选取第 3、4、7、8、11、12 个编码器块输出作为蒸馏特征，并建立如下对应关系：第 3/4 层对应 P3，第 7/8 层对应 P4，第 11/12 层对应 P5。该映射使浅层纹理细节、中层结构信息与深层语义信息能够在跨架构条件下实现分层对齐。

  - **2.2 多维特征对齐机制**

  - 针对 ViT 与 CNN 在特征表示上的异构性，本文从空间尺度与通道维度两个方面进行联合对齐。

  - （1）空间尺度对齐：以学生特征图分辨率为目标尺度，采用双线性插值对教师特征进行重采样。其中，第 3/4 层教师特征对齐至 128×128，第 7/8 层对齐至 64×64，第 11/12 层对齐至 32×32。双线性插值过程遵循标准像素映射关系：$$( [ x = p\cdot \frac{H_{src}}{H_{dst}},\quad y = q\cdot \frac{W_{src}}{W_{dst}} ] )$$并由相邻四个整数像素点加权求得目标像素值。

  - （2）通道维度对齐：在学生侧分别对 P3/P4/P5 追加 1×1 卷积，将通道数映射至教师嵌入维度 (C_t=768)。具体为：
  P3: 128×128×192 -> 128×128×768，
  P4: 64×64×384 -> 64×64×768，
  P5: 32×32×512 -> 32×32×768。
  该步骤保证教师与学生特征在通道语义空间中的一致性，为后续蒸馏损失计算提供可比基础。

  - **2.3 自适应融合蒸馏与损失函数**
  为增强教师监督信息的层级表达能力，本文对教师相邻层特征执行自适应加权融合。对于每组特征 (F_s, F_d)（对应 3/4、7/8、11/12），先在通道维拼接，再通过输出通道为 2 的 1×1 卷积生成权重映射 (F_y)，并在每个空间位置进行 softmax 归一化，得到权重 (w_1,w_2)。融合后的教师目标特征定义为：$$( [ F_{target,l}(i,j,k)=w_1(i,j)\cdot F_s(i,j,k)+w_2(i,j)\cdot F_d(i,j,k) ] [ w_1(i,j)=\frac{e^{F_y(i,j,1)}}{e^{F_y(i,j,1)}+e^{F_y(i,j,2)}},\quad w_2(i,j)=\frac{e^{F_y(i,j,2)}}{e^{F_y(i,j,1)}+e^{F_y(i,j,2)}} ] [ w_1(i,j)+w_2(i,j)=1 ])$$
  
  在损失设计上，采用 SmoothL1 约束学生对齐特征与教师融合特征之间的差异：$$( [ \mathcal{L}{feat}=\sum{l=1}^{3}\lambda_l\cdot \mathcal{L}{feat,l},\quad \mathcal{L}{feat,l}=\text{SmoothL1}(F^{aligned}{student,l},F{target,l}) ] )$$其中层级权重设置为 $$(\lambda_1=0.5,\lambda_2=0.3,\lambda_3=0.2)$$。该权重分配强调浅层细节知识迁移，以提升裂缝、孔洞等微小缺陷区域的识别能力与边界表达精度。


- **3.1 数据集与标注**
  - **3.1.1 数据预处理与切片** ：与代码 `pipeline.py` 一致，当前实现为**仅滑动窗口**切片，**不再**按小目标单独裁 ROI。全流程对**图像**与**像素级语义掩膜**同步几何变换，掩膜取值保持离散类别 id（缩放时采用最近邻，避免插值出新类别）。
  - **(1) 原图级预处理**  
    对原始高分辨率 RGB 图及其语义分割掩膜，按长宽比不变缩放，使**长边不超过**设定上限（如 3000 或 2048），得到“预处理图像”与“预处理 mask”。目的仅为控制分辨率与 IO，不强行改为方形。图像用双线性插值，掩膜用最近邻。
  - **(2) 滑动窗口 patch**  
    在预处理后的图像与 mask 上，使用固定边长窗口 $$W_{\text{patch}}\times W_{\text{patch}}$$（如 1024×1024），按步长 $$S_{\text{stride}}$$（如 512 或 800）在水平、垂直方向规则滑动；**同一窗口坐标**在图像与 mask 上同步截取，保证 patch 内像素一一对应。  
    当窗口贴近图像边界、可截取区域小于 $$W_{\text{patch}}$$ 时：先取图像与 mask 的有效重叠区域，再**对称 padding** 至 $$W_{\text{patch}}\times W_{\text{patch}}$$。实现上图像补边采用**边缘复制（edge）**，掩膜补边采用**背景类别 id** 常数填充，与训练时“背景”语义一致。
  - **(3) 训练样本筛选（可选）**  
    对得到的 $$(\text{patch\_img}, \text{patch\_mask})$$，若前景占比极低（近似全背景），可按比例**随机或均匀抽样保留**一部分，避免训练集被空背景 patch 淹没；其余窗口均作为训练/验证用的固定尺寸样本，用于学习整体构件与各类缺陷在空间上的共现关系。  
    **小目标**不再通过额外 ROI 放大采样，其监督信息来自滑窗 patch 内所包含的局部区域；若论文需强调小目标占比，仍可在 **3.1.2** 中通过统计标注框面积比等方式描述数据集特性。
  - **实现与参数** ：代码入口为 `solution/pipeline.py`，主要可调参数包括长边上限、`patch_size`、`window_stride` 及空背景保留比例等；与网络输入对齐时，取 $$H_{\text{in}}=W_{\text{in}}=W_{\text{patch}}$$（如 1024）。
